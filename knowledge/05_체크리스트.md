# 05_체크리스트

## 1. 문서 목적

이 문서는 AI 기반 안전·안심 서비스 설계 결과를 점검하기 위한 체크리스트이다.

나만의 GPT는 사용자가 안전·안심 서비스 설계, 서비스 평가, PRD 작성, 강의자료 정리, 사례 기반 분석을 요청할 때 이 체크리스트를 참고하여 빠진 항목이 없는지 확인해야 한다.

체크리스트는 다음 목적을 갖는다.

1. 위험 정의가 충분한지 확인한다.
2. 사용자와 이해관계자가 명확한지 확인한다.
3. 감지 이후 조치와 결과 확인까지 설계되었는지 확인한다.
4. 안전뿐 아니라 안심 형성 요소가 포함되었는지 확인한다.
5. AI·데이터·운영·비즈니스·윤리 요소가 균형 있게 다뤄졌는지 확인한다.
6. 서비스가 실제 현장에 적용 가능한지 평가한다.

## 2. 문제정의 체크리스트

서비스 설계의 출발점은 문제정의이다.
문제가 기술 중심으로만 정의되면 서비스 설계가 얕아질 수 있다.

| 점검 질문                                    | 확인 |
| ---------------------------------------- | -- |
| 해결하려는 위험이 명확히 정의되었는가?                    |    |
| 사고, near miss, 위험행동, 위험상태가 구분되었는가?       |    |
| 현재 대응 방식이 설명되었는가?                        |    |
| 현재 대응의 한계가 구체적으로 분석되었는가?                 |    |
| 문제를 사고 후 대응이 아니라 사고 전 위험 전이 관점에서 재정의했는가? |    |
| 안전 문제와 안심 문제가 구분되었는가?                    |    |
| 문제가 특정 기술 도입 문제로만 축소되지 않았는가?             |    |
| 사용자의 불안과 신뢰 문제가 포함되었는가?                  |    |

| 영역      | KPI 예시                       | 확인 |
| ------- | ---------------------------- | -- |
| 사고 예방   | 사고 건수 감소, near miss 감소       |    |
| 위험행동 감소 | PPE 미착용 감소, 위험구역 침범 감소       |    |

| 품질      | 오탐률, 미탐률, 알림 피로도             |    |
| 책임성     | 조치 이력 기록률, 감사 가능성            |    |

## 19. 최종 평가표

설계 결과를 간단히 평가할 때는 다음 10개 항목으로 점검한다.

| 평가 항목    | 질문                       | 점수 1~5 |
| -------- | ------------------------ | ------ |
| 문제 적합성   | 실제 중요한 안전 문제를 다루는가?      |        |
| 사용자 이해   | 보호대상자와 이해관계자가 명확한가?      |        |
| 위험 전이 분석 | 사고 전 위험상태를 분석했는가?        |        |
| 폐루프성     | 감지 이후 조치와 결과 확인이 있는가?    |        |
| AI 적합성   | AI가 맡을 역할이 명확하고 현실적인가?   |        |
| 안심 설계    | 사용자의 신뢰와 수용성을 고려했는가?     |        |
| 운영 가능성   | 현장에서 실제 운영할 수 있는가?       |        |
| 윤리성      | 개인정보, 감시, 책임 문제가 고려되었는가? |        |
| 비즈니스성    | 비용부담자와 지불 이유가 명확한가?      |        |
| 확장성      | 다른 현장이나 서비스로 확장 가능한가?    |        |

### 평가 해석

| 총점     | 해석                       |
| ------ | ------------------------ |
| 45~50점 | 매우 우수한 서비스 설계            |
| 35~44점 | 실행 가능성이 높은 설계            |
| 25~34점 | 핵심 구조는 있으나 보완 필요         |
| 15~24점 | 기능 아이디어 수준, 서비스 구조 보완 필요 |
| 14점 이하 | 문제정의부터 재검토 필요            |

## 20. 최종 점검 질문

나만의 GPT는 안전·안심 서비스 설계 결과를 마무리할 때 다음 질문을 스스로 점검해야 한다.

1. 이 서비스는 실제 위험을 줄이는가?
2. 사용자는 이 서비스를 신뢰할 수 있는가?
3. 감지 이후 조치와 결과 확인이 있는가?
4. 보호대상자를 감시가 아니라 능력증진의 대상으로 보았는가?
5. 관리자가 우선순위를 판단할 수 있는가?
6. AI 판단이 틀렸을 때 수정할 수 있는가?
7. 개인정보와 책임 문제가 관리되는가?
8. 비용을 지불할 주체와 이유가 명확한가?
9. 반복 위험을 학습하고 개선하는가?
10. 안전뿐 아니라 안심을 설계했는가?
