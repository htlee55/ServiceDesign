# 서비스 설계 산출물 템플릿

## 1. 문제정의 템플릿

현재 [현장/사용자]는 [위험상황]에 노출되어 있다.
기존 대응은 [현재 대응방식]에 의존하고 있으나,
[한계] 때문에 사고 전 위험 전이를 조기에 발견하거나
실제 안전 여부를 확인하기 어렵다.

따라서 본 서비스의 핵심 문제는
“[사용자]가 [위험상황]으로 전이되기 전에 이를 감지하고,
적절한 사람이 조치하며,
그 결과를 확인하여 사용자가 안전하다고 느낄 수 있게 하는 것”이다.

## 2. HMW 질문 템플릿

HMW 1.
어떻게 하면 [사용자/보호대상]이 위험상태로 전이되기 전에
이를 조기에 감지할 수 있을까?

HMW 2.
어떻게 하면 감지된 위험이 단순 알림에 머무르지 않고
실제 조치와 결과 확인으로 이어지게 할 수 있을까?

HMW 3.
어떻게 하면 사용자가 AI를 감시로 느끼지 않고
자신의 안전을 돕는 서비스로 받아들이게 할 수 있을까?

## 3. 서비스 구조 템플릿

서비스명:
[서비스 이름]

핵심 사용자:
[작업자 / 고령자 / 입주자 / 경비원 / 관리자 / 보호자]

핵심 가치:
[위험 감소 + 안심 형성]

주요 기능:
1. 위험 신호 수집
2. 위험 감지
3. 위험도 판단
4. 사용자별 알림
5. 조치 실행 지원
6. 조치 결과 확인
7. 반복 위험 리포트
8. 모델 성능 및 오탐 관리

폐루프 구조:
감지 → 해석 → 알림 → 조치 → 확인 → 학습 → 안심 피드백
