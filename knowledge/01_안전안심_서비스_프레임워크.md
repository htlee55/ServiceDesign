# 01_안전안심_서비스_설계_프레임워크

## 1. 문서 목적

이 문서는 AI 기반 안전·안심 서비스를 설계하기 위한 기본 프레임워크를 정리한 지식자료이다.
나만의 GPT는 이 문서를 참고하여 산업현장, 돌봄, 주거단지, 건물, 도시, 공공시설 등 다양한 안전 문제를 단순 기술 적용 과제가 아니라 서비스디자인 과제로 재정의해야 한다.

안전·안심 서비스 설계의 핵심은 위험을 감지하는 데 그치지 않고, 위험을 해석하고, 적절한 사람에게 알리고, 실제 조치와 결과 확인을 거쳐, 사용자가 안전하다고 느낄 수 있도록 신뢰를 형성하는 것이다.

---

## 2. 안전과 안심의 구분

안전과 안심은 서로 관련되어 있지만 동일하지 않다.

| 구분    | 안전                           | 안심                         |
| ----- | ---------------------------- | -------------------------- |
| 의미    | 실제 위험이 줄어든 상태                | 사용자가 위험이 관리되고 있다고 느끼는 상태   |
| 성격    | 객관적·기술적·운영적                  | 주관적·심리적·관계적                |
| 핵심 질문 | 실제 사고 가능성이 줄었는가?             | 사용자가 신뢰하고 수용할 수 있는가?       |
| 주요 지표 | 사고 감소, near miss 감소, 대응시간 단축 | 신뢰도, 불안 감소, 알림 수용률, 설명 이해도 |
| 설계 초점 | 감지, 경고, 조치, 확인               | 설명, 피드백, 개인정보 보호, 통제감      |

안전 서비스가 실제 위험을 줄이더라도 사용자가 감시당한다고 느끼거나, 알림을 이해하지 못하거나, 조치 결과를 확인하지 못하면 안심은 형성되지 않는다.

따라서 안전·안심 서비스는 항상 두 층으로 설계해야 한다.

1. 안전층: 실제 위험을 줄이는 기능
2. 안심층: 사용자가 신뢰하고 수용하게 만드는 경험

---

## 3. 안전·안심 서비스의 정의

AI 기반 안전·안심 서비스는 다음과 같이 정의한다.

“기존 CCTV, 센서, 업무 데이터, 사용자 피드백 등을 활용하여 위험상태로의 전이를 조기에 감지하고, 이를 사용자와 관리자에게 이해 가능한 방식으로 전달하며, 실제 조치와 결과 확인을 통해 위험을 줄이고, 지속적 학습과 피드백을 통해 사용자의 신뢰와 안심을 형성하는 폐루프형 서비스.”

즉 안전·안심 서비스는 단순히 위험을 감지하는 시스템이 아니다.
반드시 다음 요소를 포함해야 한다.

* 위험 신호 수집
* 위험 감지
* 위험 해석
* 사용자별 알림
* 조치 실행
* 결과 확인
* 반복 위험 분석
* 학습과 개선
* 안심 피드백

---

## 4. 위험상태 전이 모델

안전 문제는 사고가 갑자기 발생하는 것이 아니라, 정상 상태에서 위험 상태로 점진적으로 전이되는 과정으로 이해해야 한다.

기본 위험상태 전이 모델은 다음과 같다.

정상 상태 → 주의 상태 → 위험 상태 → 사고 상태 → 회복 상태

| 단계    | 의미                         | 서비스 설계 초점            |
| ----- | -------------------------- | -------------------- |
| 정상 상태 | 위험이 통제되고 일상 활동이 가능한 상태     | 안전상태 유지, 생활·작업 패턴 이해 |
| 주의 상태 | 위험 가능성이 증가하는 초기 신호가 나타난 상태 | 조기 감지, 약한 알림, 예방적 개입 |
| 위험 상태 | 즉각적인 주의나 조치가 필요한 상태        | 경고, 관리자 알림, 현장 개입    |
| 사고 상태 | 실제 피해나 사고가 발생한 상태          | 응급대응, 구조, 책임 있는 보고   |
| 회복 상태 | 조치 후 안전상태로 복귀하는 단계         | 결과 확인, 재발 방지, 안심 피드백 |

AI 안전서비스는 사고 상태만 감지하는 것이 아니라, 정상 상태에서 주의 상태, 위험 상태로 전이되는 과정을 조기에 포착해야 한다.

## 5. 폐루프형 안전서비스 구조

좋은 안전·안심 서비스는 감지에서 끝나지 않는다.
반드시 감지 이후의 조치와 결과 확인까지 포함해야 한다.

기본 폐루프 구조는 다음과 같다.

위험 신호 수집
→ 위험 감지
→ 위험 해석
→ 사용자별 알림
→ 조치 실행
→ 결과 확인
→ 학습과 개선
→ 안심 피드백

각 단계의 의미는 다음과 같다.

| 단계       | 설명                                                |
| -------- | ------------------------------------------------- |
| 위험 신호 수집 | CCTV, 센서, 위치, 출입, 작업일정, 날씨, 보고서 등에서 위험 관련 데이터를 수집 |
| 위험 감지    | AI가 위험행동, 위험상태, 이상 패턴, 장시간 무응답 등을 감지              |
| 위험 해석    | 단순 이벤트를 위험도, 원인, 맥락, 우선순위로 변환                     |
| 사용자별 알림  | 작업자, 관리자, 보호자, 경비원 등에게 상황에 맞는 방식으로 전달             |
| 조치 실행    | 회피, 경고, 순찰, 전화확인, 작업중지, 현장점검, 응급출동 등 실행           |
| 결과 확인    | 위험이 실제로 해소되었는지 확인                                 |
| 학습과 개선   | 오탐·미탐, 반복 위험, 조치 결과를 반영하여 개선                      |
| 안심 피드백   | 사용자에게 안전상태 또는 조치 완료를 알려 신뢰를 형성                    |

폐루프가 없는 안전서비스는 단순 알림 시스템에 머무를 위험이 있다.

## 6. 보호대상 Empowerment 원칙

안전·안심 서비스의 핵심 사용자는 위험에 노출된 사람이다.
산업현장에서는 작업자, 돌봄에서는 고령자, 주거단지에서는 입주자, 건물 보안에서는 이용자와 경비원이 핵심 사용자일 수 있다.

이들을 단순히 감시 대상이나 보호 대상만으로 보아서는 안 된다.
안전서비스는 보호대상자의 안전 역량을 높이는 방향으로 설계되어야 한다.

| 기존 관점       | 전환 관점                      |
| ----------- | -------------------------- |
| 작업자는 감시 대상  | 작업자는 위험을 피하고 판단하는 주체       |
| 고령자는 보호 대상  | 고령자는 가능한 한 자율성을 유지해야 하는 주체 |
| 입주자는 민원 제기자 | 입주자는 안심 경험의 핵심 사용자         |
| 경비원은 단순 대응자 | 경비원은 AI로 증강되는 현장 조치자       |
| 관리자는 감독자    | 관리자는 우선순위 판단과 개선의 책임자      |

서비스는 사용자의 자율성, 통제감, 이해 가능성을 높이는 방식으로 설계되어야 한다.

## 7. 이해관계자 구조

안전·안심 서비스는 여러 이해관계자가 함께 작동하는 서비스이다.
따라서 설계 시 항상 다음 이해관계자를 구분해야 한다.

| 이해관계자   | 역할                              |
| ------- | ------------------------------- |
| 보호대상자   | 실제 위험에 노출되는 사람                  |
| 조치자     | 위험 상황에서 직접 대응하는 사람              |
| 관리자     | 위험을 모니터링하고 자원을 배분하는 사람          |
| 비용부담자   | 서비스 비용을 지불하는 사람 또는 조직           |
| 책임자     | 사고 발생 시 책임을 지는 조직 또는 관리자        |
| 영향받는 사람 | 가족, 동료, 입주민, 방문자 등 간접 영향을 받는 사람 |
| 규제기관    | 안전 기준, 개인정보, 법적 책임과 관련된 외부 주체   |

좋은 서비스 설계는 누가 위험을 감지하고, 누가 조치하고, 누가 비용을 내고, 누가 책임지는지를 명확히 해야 한다.

## 8. AI의 역할

AI는 안전서비스에서 사람을 대체하는 것이 아니라 사람의 안전 역량을 증강하는 역할을 해야 한다.

AI의 주요 역할은 다음과 같다.

| 역할    | 설명                               |
| ----- | -------------------------------- |
| 감지    | 사람이 놓치기 쉬운 위험행동, 위험상태, 이상 패턴을 감지 |
| 해석    | 감지된 이벤트를 위험도, 원인, 맥락, 우선순위로 변환   |
| 예측    | 사고 전 위험 전이 가능성을 예측               |
| 우선순위화 | 제한된 인력과 자원을 어디에 먼저 투입할지 추천       |
| 설명    | 왜 위험하다고 판단했는지 사람이 이해할 수 있도록 제시   |
| 학습    | 오탐, 미탐, 조치 결과, 현장 변화를 반영하여 지속 개선 |
| 기록    | 위험 이벤트, 조치 이력, 판단 근거를 감사 가능하게 저장 |

AI는 최종 책임자가 아니라 의사결정 지원자이다.
고위험 상황에서는 사람이 최종 판단과 책임을 갖는 구조가 필요하다.

## 9. 안전층과 안심층

안전·안심 서비스는 안전층과 안심층을 분리해 설계해야 한다.

### 9.1 안전층

안전층은 실제 위험을 줄이는 기능이다.

예시:

* 위험행동 감지
* 위험구역 침범 감지
* 낙상 또는 장시간 무응답 감지
* 보행자-차량 근접 위험 감지
* 침입 또는 이상 출입 감지
* 경고 및 현장 알림
* 관리자 조치 요청
* 조치 결과 확인
* 반복 위험 분석

### 9.2 안심층

안심층은 사용자가 서비스를 신뢰하고 수용하도록 만드는 기능이다.

예시:

* 왜 알림이 발생했는지 설명
* 개인정보 보호와 익명화
* 감시 목적이 아니라 보호 목적임을 명확히 함
* 조치 완료 알림
* 사용자의 피드백 반영
* 오탐 정정 기능
* 보호자 또는 입주민에게 안전상태를 알려주는 안심 피드백
* 사용자가 불필요한 불안을 느끼지 않도록 알림 수준 조절

안전층 없이 안심만 있으면 실질적 보호가 약하고,
안심층 없이 안전만 있으면 사용자가 감시와 통제로 느낄 수 있다.

## 10. 안심 형성요인

안심은 단순히 “위험이 없다”는 말로 형성되지 않는다.
사용자가 위험 관리 과정을 이해하고 신뢰할 수 있어야 한다.

안심 형성의 주요 요인은 다음과 같다.

| 요인      | 설명                             |
| ------- | ------------------------------ |
| 설명가능성   | 왜 위험으로 판단했는지 이해할 수 있어야 함       |
| 통제감     | 사용자가 알림을 확인하거나 피드백할 수 있어야 함    |
| 개인정보 보호 | 감시당한다는 불안을 줄여야 함               |
| 조치 확인   | 위험이 실제로 해소되었는지 알려야 함           |
| 일관성     | 알림과 조치 기준이 예측 가능해야 함           |
| 과잉알림 방지 | 불필요한 알림으로 신뢰를 떨어뜨리지 않아야 함      |
| 책임 명확성  | 누가 조치하고 책임지는지 분명해야 함           |
| 참여 가능성  | 현장 사용자와 보호대상자가 개선에 참여할 수 있어야 함 |

## 11. 윤리·개인정보·책임 원칙

AI 기반 안전·안심 서비스는 감시와 보호의 경계에 있기 때문에 윤리적 설계가 중요하다.

반드시 고려해야 할 원칙은 다음과 같다.

1. 목적 제한
   수집된 데이터는 안전과 안심 목적에 맞게 사용해야 한다.

2. 최소 수집
   필요한 데이터만 수집하고, 불필요한 개인식별 정보는 줄인다.

3. 익명화와 비식별화
   얼굴, 이름, 개인 행동 이력 등 민감한 정보는 가능한 한 익명화한다.

4. 설명가능성
   AI 판단의 이유와 기준을 사용자가 이해할 수 있어야 한다.

5. 사람의 최종 판단
   고위험 조치, 징계, 처벌, 책임 판단은 AI 단독으로 하지 않는다.

6. 오탐·미탐 관리
   AI의 오류를 인정하고 수정할 수 있는 피드백 구조를 갖춘다.

7. 책임 소재 명확화
   알림을 받은 사람, 조치한 사람, 운영자, 서비스 제공자의 책임을 구분한다.

8. 신뢰 기반 운영
   사용자를 감시 대상으로 대하지 않고, 안전 역량을 높이는 동반자로 대한다.

## 12. 서비스 가치와 KPI

안전·안심 서비스의 가치는 단순히 사고 감소만으로 설명해서는 부족하다.
고객과 현장 이해관계자가 체감할 수 있는 다양한 KPI를 설계해야 한다.

| 가치 영역  | KPI 예시                          |
| ------ | ------------------------------- |
| 사고 예방  | 사고 건수 감소, near miss 감소, 위험행동 감소 |
| 대응 향상  | 감지 후 알림 시간, 조치 완료 시간, 응급대응 시간   |
| 운영 효율  | 순찰 효율, 보고 자동화, 관리업무 시간 절감       |
| 교육 효과  | 반복 위험행동 감소, 교육 후 개선률            |
| 신뢰와 안심 | 사용자 신뢰도, 불안 감소, 알림 수용률          |
| 규제 대응  | 조치 이력, 감사 기록, 컴플라이언스 보고         |
| 비용 효과  | 사고 비용 감소, 보험료 절감 가능성, 관리비 절감    |

## 13. 안전·안심 서비스 설계 원칙

나만의 GPT는 안전·안심 서비스를 설계할 때 다음 원칙을 항상 적용한다.

1. 사고 후 대응이 아니라 사고 전 위험 전이를 감지한다.
2. 위험행동뿐 아니라 위험상태와 위험 맥락을 함께 본다.
3. AI 감지는 알림에서 끝나지 않고 조치와 결과 확인으로 이어져야 한다.
4. 보호대상자는 감시 대상이 아니라 능력증진의 대상이다.
5. 관리자는 단순 대시보드 사용자가 아니라 우선순위 판단과 개입의 주체이다.
6. 서비스는 안전뿐 아니라 안심을 함께 설계해야 한다.
7. 개인정보 보호, 설명가능성, 피드백, 책임 소재를 명확히 해야 한다.
8. 모델은 배포 후에도 오탐, 미탐, drift, 현장 변화에 따라 지속 관리되어야 한다.
9. 기존 CCTV, 센서, EHS, 시설관리, 출입관리, 업무관리 시스템과 연동 가능성을 고려한다.
10. 서비스 가치는 사고 감소뿐 아니라 관리비 절감, 보험, 규제 대응, 교육, 운영 효율과 연결해 설명한다.

## 14. 기본 설계 질문

안전·안심 서비스 설계 시 항상 다음 질문을 확인한다.

1. 무엇이 위험인가?
2. 위험은 어떤 상태 전이를 거쳐 사고로 이어지는가?
3. 누가 위험에 노출되어 있는가?
4. 누가 가장 먼저 알아야 하는가?
5. 누가 실제로 조치할 수 있는가?
6. 조치가 완료되었는지 어떻게 확인할 것인가?
7. 사용자는 이 서비스를 보호로 느낄 것인가, 감시로 느낄 것인가?
8. AI 판단이 틀렸을 때 어떻게 수정할 것인가?
9. 어떤 데이터가 필요하며 개인정보 위험은 무엇인가?
10. 어떤 KPI로 서비스 성과를 평가할 것인가?

## 15. 최종 지향점

AI 기반 안전·안심 서비스의 최종 지향점은 다음과 같다.

기존 CCTV, 센서, 업무 데이터를 활용해 위험 전이를 조기에 감지하고,
사용자별로 이해 가능한 방식으로 알림을 제공하며,
실제 조치와 결과 확인을 통해 위험을 줄이고,
지속적인 학습과 설명가능성을 통해 사용자의 신뢰와 안심을 형성하는 서비스.

즉, 안전·안심 서비스는 감시 시스템이 아니라
사람의 안전 역량을 증강하고 현장의 회복탄력성을 높이는 폐루프형 운영 서비스이다.
